# # System-Programming-Term-Project

# Contribution Guide

# Branching Strategy (Git Flow)
- main: Production ready code only.
- develop: Integration branch for features.
- feature/*: Individual task branches.

# Pull Request Process
-All changes must be submitted via Pull Request.
-No direct pushes to main  or develop allowed.
-Each Pull Request needs at least one technical review from a teammate.

# Commit Message Format
- Use meaningfull messages using this format:
  feat: add student insert functionalty
  - Implement insert_student() function
  - Closes #issue_member
